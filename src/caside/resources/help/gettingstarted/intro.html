Introduction to Agility SipServlet IDE

The emergence of next generation networks will introduce converged applications that drive service provider infrastructure deployments and enhance revenue. Since the conversion of IP communication is escalating on global scale there is a need for swift, commercial development and deployment of application and services. BayPackets (BP) AgilityTM SipServlet Integrated Development Environment (IDE) provides an integrated environment for development, deployment and debugging of Session Initiation Protocol/Hyper Text Transfer Protocol (SIP/HTTP) based services.

Agnity IDE is a plugin, it conforms to the Eclipse Plug-in Architecture. Eclipse is responsible for booting and running the platform and plug-in declaration and management. It also provides basic mechanism of extensibility so that new plug-ins can add new processing elements to existing plug-ins. With the help of this IDE user can efficiently develop and deploy SIP based services. IDE also provides full functionality related to deployment and debugging of these services. User can attach this IDE to a already running Agnity SIP Application Server and can deploy/debug the services. The main objective is to make the user more comfortable in developing SIP/HTTP based services.

Bay Packets Services are of three major types:

  • Sesion Initiation Protocol(SIP) based services.
  • Hyper Text Transfer Protocol (HTTP) based services.
  • Converged Services (SIP as well as HTTP both).

BayPackets  provides Service building blocks (SBB) for SIP based services. IDE also provides SBB Event Listener wizard
