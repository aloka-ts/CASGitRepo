Service Management

Service Management deals with the development, deployment, starting, activating, deactivating, stopping and undeployment of service on the Agnity SIP Application Server.

This section will explain you how to:

Ø      Develop Service

Ø       Viewing Service Information

Ø       Deploy a Service

Ø       Start a Service

Ø       Activate a Service

Ø       Deactivate a Service

Ø       Stop a Service

Ø       Undeploy a Service

Before the user can move ahead, s/he needs to understand the Service Management interface.

 

 

Menu

Click this button to view the menu list.

Load Service On

Click this button to switch on the auto load service feature. This is a very important feature. When this feature is on the IDE forks a process that pings the CAS every 1 second (so that it can know the status of all the services deployed on CAS). This feature can be switched off by clicking again that button. By default this feature is off. Before interacting with this Service Management view (This feature should be switched on).

CAS Info

Click this button to get the Information (address) of the CAS attached to IDE.(or started by IDE).

Stop CAS

Click this button to stop the CAS instance (Gracefull shut down). This is the status button. It show whether the CAS related to this Service Management View is running or not.  It changes its color on the basis of CAS status. If the color is red it means CAS is nor running and if the color is green that means CAS is running.

Workspace

The workspace is the rectangular space where all the information is displayed based on your action.
